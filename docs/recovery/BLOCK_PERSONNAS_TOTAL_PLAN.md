# Bloc Personnas Total Plan

## Objectif

Faire de la page `personnas` une vraie porte d'entree pedagogique vers les entretiens, tout en permettant la creation libre de nouveaux personnas.

Le bloc doit relier chaque persona a :
- sa fiche
- son historique
- sa grille d'entretien
- son prompt

## Etape 1. Cadrage du bloc

But :
- fixer la structure cible du bloc
- decider les actions par persona
- decider la logique de la fiche persona

Resultat attendu :
- on sait exactement ce que doit contenir une carte et une fiche persona

## Etape 2. Refaire la page liste `personnas`

Contenu :
- titre / sous-titre clairs
- recherche
- filtres visibles
- hierarchie plus propre
- cartes plus lisibles

Resultat attendu :
- une page plus proche des captures et plus comprehensible

## Etape 3. Refaire les cartes persona

Chaque carte devrait afficher :
- nom
- description courte
- statut
- auteur ou type si utile
- actions principales

Actions a garder :
- `Commencer`
- `Historique`
- `Voir la fiche`
- `Prompt` ou `Modifier` selon le role

Resultat attendu :
- chaque persona devient un vrai point d'entree de travail

## Etape 4. Creer la logique de fiche persona

La fiche persona doit regrouper :
- identite du persona
- resume
- grille d'entretien associee
- prompt associe
- points de vigilance / conseils si besoin
- bouton `Commencer un entretien`

Important :
- on lie grille + prompt
- mais on ne les fusionne pas dans un seul bloc opaque

Resultat attendu :
- chaque persona possede un espace coherent

## Etape 5. Verifier / ameliorer le flux de creation

Contenu :
- garder la creation libre
- verifier la lisibilite des 3 colonnes
- rendre plus clair :
  - nom
  - description
  - prompt
  - validation Cauldron
- rendre la creation plus rassurante

Resultat attendu :
- creation simple mais credible

## Etape 6. Verifier l'edition / historique

Contenu :
- historique par persona
- acces facile depuis la carte
- edition du prompt si autorisee
- continuite entre fiche, historique et entretien

Resultat attendu :
- le persona n'est pas juste cree, il est reutilisable et suivi

## Etape 7. Preparer le lien avec l'accueil

Contenu :
- choisir 2 ou 3 personnas exemples
- preparer leur remontee future sur la home
- sans refaire encore la home

Resultat attendu :
- le bloc persona est pret a alimenter l'accueil

## Definition du bloc personnas termine

Le bloc sera assez fini quand on aura :
1. une page liste claire et propre
2. des cartes utiles
3. historique par persona
4. fiche persona complete
5. creation / edition coherentes
6. grille + prompt bien relies
7. base prete pour afficher quelques personnas sur l'accueil

## Ordre recommande

1. cadrage
2. page liste
3. cartes
4. fiche persona
5. creation / edition
6. historique
7. lien futur avec home

## Recommandation pour la prochaine reprise

Commencer par :
- cadrage precis de la fiche persona
- puis refonte de la page liste `personnas`

Pourquoi :
- ce sont les deux briques qui organisent tout le reste
